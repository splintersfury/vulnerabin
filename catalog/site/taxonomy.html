Source taxonomy v2

62 source classes across 11 groups. The canonical reference is taxonomy/binary/sources_comprehensive.md; the detection-checklist + defense library is at taxonomy/binary/defense_library.json.

F-001 F
NTFS-junction-write-by-elevated-process
Defense: Apply protected DACL to every ancestor the binary creates; open paths with FILE_OPEN_REPARSE_POINT/OBJ_DONT_REPARSE; refuse if FILE_ATTRIBUTE_REPARSE_POINT...
F-002 F
NTFS-junction-read-by-elevated-process
Defense: Same as F-001 plus HMAC/sign config files so even if read succeeds, content is rejected unless the signature checks.
F-003 F
NTFS-hardlink-substitution
Defense: Open with FILE_FLAG_OPEN_REPARSE_POINT, verify nNumberOfLinks == 1 via GetFileInformationByHandle; or use FILE_DISPOSITION_POSIX_SEMANTICS for delete operations.
F-004 F
Symlink-by-low-priv-into-protected-target (Linux)
Defense: Use O_NOFOLLOW; openat() with AT_SYMLINK_NOFOLLOW; mkstemp for temp files instead of predictable paths.
F-005 F
World-writable IPC socket
Defense: Linux: socket mode 0700 / per-user; SO_PEERCRED to verify caller UID. Windows: explicit DACL granting only owning service account.
F-006 F
World-writable / user-readable config / log file
Defense: Apply protected DACL on file creation; rotate logs to admin-only directories; redact sensitive content before logging.
F-007 F
File-association / drag-drop trust handler
Defense: Treat file content as untrusted; sanitize path components; drop privileges before parsing; use a sandbox process for file-content parsing.
I-001 I
Named-pipe-unauthenticated-read
Defense: Explicit SDDL grant only to the owning service principal; authenticate caller via ImpersonateNamedPipeClient + token check.
I-002 I
Named-pipe-unauthenticated-write
Defense: Explicit SDDL; authenticate caller via ImpersonateNamedPipeClient + group membership check, OR GetNamedPipeClientProcessId + image-path allowlist.
I-003 I
D-Bus method caller-uid trust
Defense: Always read caller uid from g_dbus_method_invocation_get_sender(invocation) then bus.get_unix_user(sender). Discard any uid argument.
I-004 I
COM elevation moniker / IElevator interface
Defense: Restrict LaunchPermission to admin-only SIDs. Authenticate caller in every method via CoImpersonateClient + OpenThreadToken + token check.
I-005 I
WM_COPYDATA / window message
Defense: ChangeWindowMessageFilter / ChangeWindowMessageFilterEx to block specific messages from non-elevated; rely on UIPI default.
I-006 I
ALPC port (kernel)
Defense: Implement an ALPC connection callback that validates caller via AlpcGetMessageAttribute + PsLookupProcessByProcessId checks.
I-007 I
Mailslot
Defense: Set DACL via lpSecurityAttributes; legacy IPC, prefer migrating to named pipes.
I-008 I
Shared section / cross-process memory
Defense: Per-session local namespace (\Sessions\<sid>\BaseNamedObjects\); explicit DACL.
I-009 I
Localhost TCP IPC (port-bound)
Defense: Application-level auth: TLS with mutual-cert, HMAC over messages, OAuth-style tokens. The IPC layer offers no protection.
N-001 N
DNS wire-format parser
Defense: Strict bounds checking on every length field; compression-pointer cycle detection; explicit max-iterations on RR-list traversal.
N-002 N
HTTP listener (HTTP.sys / IIS / custom)
Defense: Use battle-tested HTTP libraries; strict RFC compliance; explicit length checks; reject malformed Transfer-Encoding combinations.
N-003 N
TLS / network protocol parser (transport-level)
Defense: Cert pinning; HMAC over payloads; protocol parser fuzzing.
N-004 N
Custom application protocol (payload-level)
Defense: Use schema-driven parsers (protobuf, capnproto); explicit type tags; bounds-checked TLV walks.
N-005 N
SSRF
Defense: URL allowlist; DNS resolution before fetch + filter against internal IP ranges; disable redirects; disable file://, gopher://.
N-006 N
WebSocket
Defense: Origin check on upgrade; per-message length checks; auth token in upgrade or first message.
N-007 N
Multicast / mDNS / broadcast listener
Defense: Treat multicast/broadcast packets as untrusted by definition; rate-limit; sanity-check packet structure aggressively.
K-001 K
IOCTL input buffer (IRP_MJ_DEVICE_CONTROL)
Defense: IoCreateDeviceSecure with admin-only SDDL; SeAccessCheck per IOCTL; METHOD_BUFFERED default; ProbeForRead/ProbeForWrite for METHOD_NEITHER inside...
K-002 K
FSCTL / minifilter port operation
Defense: SDDL on the port; per-message authentication; bounds-check input/output sizes; FltGetFilterFromName ACL check.
K-003 K
WMI provider input
Defense: Validate every WMI method input; refuse callers without admin token.
K-004 K
Object/process/image-load/registry callback
Defense: Callbacks should be defensive (validate, return STATUS_ACCESS_DENIED on suspect ops); never operate on user data without validation.
K-005 K
Non-IOCTL IRP (read/write/create)
Defense: Same as K-001 for the relevant IRP_MJ_*
K-006 K
Kernel-resident user-mapped buffer (multi-fetch / Cc-mapped)
Defense: Kernel-pool snapshot before validate-and-use: copy user buffer to kernel-allocated buffer first, then validate-and-operate on the kernel copy.
U-001 U
Elevated process argv parsing
Defense: Don't trust argv; for service/installer use case, derive args from authoritative source (registry under admin DACL, SCM-set ImagePath).
U-002 U
Environment variable trust
Defense: Sanitize/whitelist env vars at process boundary; for setuid/elevated, use sanitized libc env routines or strip non-essential vars.
U-003 U
Custom-protocol-handler URL routing
Defense: Strict URL parser; reject unexpected URL components; treat URL as untrusted input regardless of source.
U-004 U
Clipboard / drag-drop content
Defense: Don't trust clipboard content for privileged operations; validate format before parsing.
U-005 U
Window-message input
Defense: ChangeWindowMessageFilter to block privileged messages from non-elevated; UIPI default protection.
T-001 T
WinVerifyTrust on file path (process-hollowing)
Defense: Verify in-memory code: hash caller's image base via ZwReadVirtualMemory and compare against expected hash; or use PPL on the trusting service.
T-002 T
PEB.ImagePathName trust
Defense: Use NtQueryInformationProcess(ProcessImageFileName, class 29) which sources from EPROCESS->SectionObject (kernel-authoritative); never read from PEB for...
T-003 T
Token / SID / membership check spoofable
Defense: Use authoritative kernel-derived checks; for cross-process trust, prefer ImpersonateLoggedOnUser + SeAccessCheck against a kernel object.
T-004 T
Caller-process-image-path trust without impersonation
Defense: Don't have IPC at all, OR require fresh authentication beyond image-path identity (HMAC token, mutual TLS, signed session ticket).
T-005 T
Path-traversal / canonicalization in trust check
Defense: Canonicalize before compare: GetFullPathNameW(path, MAX_PATH, canonical, NULL) then wcsncmp; or PathIsPrefix from shlwapi on canonicalized inputs.
T-006 T
Same-sign / co-signing assumption
Defense: Don't trust co-signing as auth. Use mutual TLS, fresh tokens, or kernel-mode authentication primitives.
T-007 T
Internal-component-as-untrusted-source
Defense: Threshold-protocol design assuming K-of-N components are honest; cross-validate aggregations; require independent signatures from at least M components.
UP-001 UP
Auto-updater unsigned-binary execution
Defense: WinVerifyTrust on every downloaded binary before exec; signature pinning with backup keys; refuse exec on signature failure.
UP-002 UP
Squirrel.Windows update channel
Defense: Restrict packages directory to admin-write; sign all packages; verify package signatures before unpack/install.
UP-003 UP
electron-updater feed manipulation
Defense: HTTPS feed; sign latest.yml / app-update.yml; cert pinning; code-sign on the downloaded package.
UP-004 UP
MSI custom-action argv
Defense: Don't trust MSI properties for security decisions; restrict who can msiexec /i with custom properties.
UP-005 UP
Installer extract-then-execute
Defense: Extract to admin-write-only path; canonicalize before exec; verify signature on extracted binary before running.
C-001 C
Registry value with permissive DACL trusted by elevated
Defense: Admin-only SDDL on the subkey; for HKCU values used by elevated process, validate per-user-context they don't influence cross-user decisions.
C-002 C
Config file in user-writable location
Defense: Variant of F-002/F-006 — same defense.
C-003 C
GPO / policy preference
Defense: Sign / encrypt sensitive values; rotate keys; use modern Group Policy Preferences APIs that don't store credentials in cpassword.
C-004 C
Service start parameters
Defense: Restrict SERVICE_START to admins; don't accept lpServiceArgVectors from non-admin callers.
E-001 E
ipcMain.on/handle channel
Defense: contextIsolation: true; validate every handler argument; never pass renderer-supplied args directly to child_process.exec / fs.writeFile / eval; use a...
E-002 E
Custom protocol handler URL routing (Electron)
Defense: Strict URL parser; reject unexpected paths; treat URL as untrusted input.
E-003 E
webPreferences.nodeIntegration:true
Defense: contextIsolation: true; nodeIntegration: false (default modern Electron); use contextBridge.exposeInMainWorld for explicit API surface.
E-004 E
protocol.registerFileProtocol unvalidated
Defense: Strict URL → path mapping; reject paths with .. or absolute paths; canonicalize and prefix-check against an expected base.
E-005 E
BrowserWindow.loadURL with attacker URL
Defense: Allowlist origins for loadURL; strip query/fragment from user-supplied URLs; never disable webSecurity.
E-006 E
Renderer XSS in trusted origin
Defense: DOMPurify on user-rendered content; CSP with strict-dynamic; nonce-based script loading.
E-007 E
Squirrel/electron-updater feed (Electron-specific overlap UP-003)
Defense: Same as UP-003.
W-001 W
Authenticated user session as source
Defense: Tenant-scope every request; explicit authorization check on every endpoint; per-row access control.
W-002 W
Cross-tenant authorization gap
Defense: Tenant ID required on every query; SQL/ORM scope filters set by middleware not handler.
W-003 W
Pre-auth endpoint
Defense: Auth required by default; explicit allowlist for genuinely public endpoints; rate-limit.
CR-001 CR
Constant-key / many-time-pad encryption of privileged data
Defense: BCryptGenRandom for keys; nonces from BCryptGenRandom per-message; AEAD modes (GCM/CCM) which fail catastrophically on nonce reuse.
CR-002 CR
Reserved (signature/HMAC omitted by default)
Defense: (placeholder) Sign/HMAC every privileged datum.
